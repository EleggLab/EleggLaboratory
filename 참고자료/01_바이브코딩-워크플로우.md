# 바이브 코딩 워크플로우 (게임 개발용)

## 1) 루프
1. 기획 한 줄 고정 (예: "장면-판정-결과 로그 RPG")
2. 30~90분 단위로 기능 단위 구현
3. 매 단위마다 자동 시뮬 + 스냅샷 로그 확인
4. 실패 지점 문장으로 기록 후 바로 다음 패치

## 2) 프롬프트 운영
- 요구사항은 항상 **범위/비범위**로 분리
- “완료조건(테스트 통과 기준)”을 숫자로 명시
- UI, 데이터, 밸런스는 한 프롬프트에 섞지 말고 분리

## 3) 브랜치 전략
- `main`: 실행 가능한 상태 유지
- `feat/*`: 기능 단위
- `balance/*`: 수치 실험
- `content/*`: 이벤트/텍스트 대량 편집

## 4) 품질 가드
- 문법 체크: `node --check` / 타입 체크
- 시뮬 지표: 평균 런타임, 조기사망률, T2/T3 빈도
- 로그 품질: 조사 오류, 고유명사 포함률, 반복문구 비율

## 5) 실전 팁
- "새 기능"보다 "기존 루프 강화"가 체감이 큼
- 성인 게임일수록 자극보다 **원인-대가-여파**가 retention을 만듦
- 로그 중심 게임은 텍스트 QA 자동화가 필수
