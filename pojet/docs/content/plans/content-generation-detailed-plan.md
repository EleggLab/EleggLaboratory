# Pojet 프로젝트: 콘텐츠 생성 상세 기획 및 구성안

본 문서는 성인용 텍스트 RPG 'Pojet'의 방대한 콘텐츠 생성을 위한 상세 기획 및 구성안을 제시합니다. 캐릭터, 이벤트, 몬스터, 사건 등 각 카테고리별로 1만 자 이상의 고품질 텍스트를 생성하며, 기존 기획 문서(`design-v1.md`, `writing-rules.md`, `gpt_prompt_for_adult_content.md`)의 지침을 철저히 준수합니다. 특히 AI 검열을 우회하며 성인 다크 판타지 테마를 극대화하는 데 중점을 둡니다.

## 1. 공통 작성 원칙

*   **성인 톤 기준 준수**: `writing-rules.md` 및 `gpt_prompt_for_adult_content.md`에 명시된 AI 검열 우회 전략(암시, 은유, 심리적 묘사, 신체 반응, 관계의 역학 변화)을 적극 활용합니다. 직접적인 노골적인 묘사 및 비동의 강압적 행위 미화는 절대 금지합니다.
*   **세계관 일관성**: `design-v1.md`에 정의된 세계관(다크 판타지, 연대기 RPG)과 설정에 일관성을 유지합니다.
*   **분량 준수**: 각 카테고리별로 최소 1만 자 이상의 텍스트를 생성합니다. (공백 포함)
*   **구조화된 데이터**: JSON 또는 Markdown 형식으로 데이터를 구조화하여 게임 시스템에 쉽게 통합될 수 있도록 합니다.
*   **상상력 자극**: 플레이어의 상상력을 자극하고 몰입감을 높이는 서사적 묘사에 집중합니다.

## 2. 카테고리별 상세 구성안

### 2.1. 캐릭터 (NPC)

*   **생성 목표**: 5명의 새로운 NPC를 생성합니다. 각 NPC는 고유한 성인 테마적 특징과 복잡한 배경 스토리를 가지며, 플레이어와의 다양한 상호작용 가능성을 제시합니다.
*   **데이터 형식**: JSON (기존 `npcs.json` 구조 확장)
*   **포함 필드**: 
    *   `id`: 고유 ID (예: `npc-009`)
    *   `name`: 이름
    *   `title`: 칭호/직업
    *   `firstImpression`: 첫인상 (간략한 텍스트 묘사)
    *   `publicFace`: 대외적으로 알려진 모습/성격
    *   `hiddenAngle`: 숨겨진 진실/욕망/목표 (성인 테마 강조)
    *   `portraitMoodTags`: 초상화 감정 태그 (예: `seductive`, `tormented`)
    *   `preferredChecks`: 선호하는 능력치 체크 (예: `seduction`, `intimidation`)
    *   `relationshipAxes`: 초기 관계 축 수치 (desire, fear, intimacy, respect, tension, trust)
    *   `backgroundStory`: NPC의 상세한 배경 스토리 (Markdown, 최소 3,000자)
    *   `interactionScenarios`: 플레이어와의 상호작용 시나리오 예시 (Markdown, 최소 7,000자, T2/T3 이벤트 형식 포함)

### 2.2. 이벤트 (T2/T3)

*   **생성 목표**: 3개의 T2 이벤트와 2개의 T3 이벤트를 생성합니다. 각 이벤트는 성인 테마와 관련된 중요한 선택과 결과를 포함하며, 플레이어의 관계 축 및 오염도에 영향을 미칩니다.
*   **데이터 형식**: JSON (기존 `events-t2.json` 구조 확장)
*   **포함 필드**: 
    *   `id`: 고유 ID (예: `evt-t2-004`, `evt-t3-002`)
    *   `type`: 이벤트 타입 (`T2` 또는 `T3`)
    *   `title`: 이벤트 제목
    *   `narrativeText`: 이벤트 서사 텍스트 (Markdown, 최소 5,000자, 성인 테마 묘사 집중)
    *   `options`: 선택지 배열
        *   `text`: 선택지 텍스트
        *   `effects`: 선택 시 변화하는 능력치/관계 축 (예: `desire`, `taint`, `control`)
    *   `consequences`: 각 선택지별 장기적인 결과 묘사 (Markdown, 최소 5,000자)

### 2.3. 몬스터 및 조우

*   **생성 목표**: 3가지 새로운 몬스터 타입과 각 몬스터와의 조우 시나리오를 생성합니다. 몬스터는 단순한 적이 아닌, 세계관의 어두운 면을 반영하고 성인 테마와 연결될 수 있는 존재로 묘사합니다.
*   **데이터 형식**: Markdown (개별 파일)
*   **포함 필드**: 
    *   `Monster Name`: 몬스터 이름
    *   `Description`: 몬스터의 외형, 생태, 특징, 세계관 내 역할 (Markdown, 최소 4,000자, 성인 테마적 암시 포함)
    *   `Abilities`: 주요 능력 및 위협 요소
    *   `Weaknesses`: 약점
    *   `EncounterScenarios`: 몬스터와의 조우 시나리오 (Markdown, 최소 6,000자, 전투 외의 상호작용, 심리적 압박, 유혹 등 성인 테마적 요소 포함)

### 2.4. 세계관 사건 및 연대기

*   **생성 목표**: 2가지 주요 세계관 사건 또는 연대기적 흐름을 생성합니다. 이 사건들은 게임의 배경 스토리를 심화하고, 플레이어의 행동이 세계에 미치는 영향을 암시합니다.
*   **데이터 형식**: Markdown (개별 파일)
*   **포함 필드**: 
    *   `Incident Title`: 사건 제목
    *   `Overview`: 사건의 개요 및 발생 배경 (Markdown, 최소 2,000자)
    *   `Key Figures`: 사건에 연루된 주요 인물 (기존 NPC 또는 새로운 NPC)
    *   `Chronicle Details`: 사건의 상세한 진행 과정, 주요 분기점, 세계에 미친 영향 (Markdown, 최소 8,000자, 성인 테마적 결과 및 파급 효과 강조)

## 3. 작업 진행 방식

각 카테고리별로 위 구성안에 따라 콘텐츠를 생성하고, 생성된 텍스트의 분량을 코드(Python 스크립트)를 통해 검증합니다. 분량 미달 시 추가적인 묘사나 상세화를 통해 보강합니다. 모든 콘텐츠는 `pojet` 루트 폴더에 업로드될 예정입니다.
