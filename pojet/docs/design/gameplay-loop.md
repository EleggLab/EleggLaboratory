# Gameplay Loop

## 루프 요약
1. 런 시작/캐릭터 생성
2. 자동 틱 진행 (탐험/사회/전투/휴식)
3. 보상/피해/관계/평판 반영
4. 이벤트 디스패치(T0~T3)
5. T2는 제한시간 후 프리셋 자동 처리
6. T3는 강제 정지+플레이어 선택
7. 퀘스트/막 진행, 레벨업, 연대기 기록

## 틱 처리 우선순위
- phase resolve -> checks -> rewards -> event dispatch -> state transition -> ui sync

## 자동/수동 경계
- 자동: 이동, 일반 전투, 일반 루팅, 반복 의뢰
- 수동: 세력, 서브클래스, 유물, 계승, 관계 확정

## 캐릭터 커스터마이징 및 동적 외형 변화 반영

플레이어 캐릭터 및 NPC의 외형 변화는 게임 루프 내에서 다음 지점에서 반영되어 서사적 깊이와 몰입감을 더한다.

1.  **런 시작/캐릭터 생성**: 초기 캐릭터 생성 시 설정된 외형 특징(텍스트 기반)이 게임 시작 시점에 반영되며, 이후 모든 텍스트 묘사의 기본값이 된다.
2.  **보상/피해/관계/평판 반영**: 
    *   **보상**: 특정 의상, 장신구, 혹은 신체 변이를 유발하는 아이템 획득 시, 캐릭터의 외형 묘사 텍스트가 업데이트되고, 시각적 에셋(레이어드 스프라이트/초상화)이 동적으로 변경될 수 있다.
    *   **피해**: 전투 또는 이벤트로 인한 신체적 피해(상처, 훼손)는 외형 묘사 텍스트에 반영되며, `visualStateTags`를 통해 시각적으로 표현될 수 있다.
    *   **관계/평판**: NPC와의 관계 변화나 평판(명성/악명) 변화는 캐릭터의 표정, 자세, 의상 등 외형 묘사에 영향을 미칠 수 있다. (예: 악명이 높아질수록 더 거칠거나 퇴폐적인 의상으로 변화)
3.  **이벤트 디스패치(T0~T3)**: 
    *   **T2/T3 이벤트**: 핵심 이벤트 발생 시, 플레이어의 선택이나 이벤트 결과에 따라 `maturityTags`, `relationshipAxes`, `오염(Taint)` 수치가 변화하고, 이에 따라 캐릭터의 외형 묘사 텍스트가 동적으로 변경된다. 특히 성인 테마와 관련된 이벤트에서는 암시적이고 심리적인 외형 변화 묘사가 강조된다.
    *   **신체 변이**: 특정 이벤트는 캐릭터의 신체 변이를 유발할 수 있으며, 이는 텍스트 묘사와 함께 시각적 에셋(부분 일러스트/CG)으로 강조될 수 있다.
4.  **UI Sync**: 캐릭터의 외형 변화는 UI의 초상화 및 관련 정보 창에 실시간으로 동기화되어 플레이어에게 시각적인 피드백을 제공한다.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 구현팀의 루프 컨트롤러 기본 순서
- QA의 자동/수동 경계 검증 기준
