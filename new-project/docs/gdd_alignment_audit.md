# GDD Alignment Audit

검사 기준: `game_design_document.md`의 3.1, 13.1, 13.2

## 확인된 불일치와 수정 결과

1. 캐릭터 정체성 입력 순서 불일치
- 기준: 13.2 마지막 문단(기억 편린 이후 이름/성별/외모 결정)
- 이전: 게임 시작 직후 이름/성별/외모 입력
- 수정: 모든 선택 완료 후 정체성 입력하도록 순서 변경

2. 시작 장비 결정 누락
- 기준: 13.2(선택을 통해 시작 장비 결정)
- 이전: 장비 시스템 없음
- 수정: 선택 결과(오프닝/생존 방식/상처)를 조합해 시작 장비 자동 지급

3. 외모 입력 방식 불일치
- 기준: 13.2(외모 텍스트 묘사 선택)
- 이전: 자유 텍스트만 입력
- 수정: 텍스트 묘사 프리셋 선택 + 직접 입력 옵션 추가

4. 저장 포맷 오염
- 기준: 캐릭터 시트 저장 의도
- 이전: 생존 스탯 내부 제어 필드(`bounds`)까지 저장
- 수정: 저장용 `survival` 직렬화에서 게임 데이터 필드만 저장

5. 스킬 정의 일부 누락
- 기준: 3.1.4(성애 스킬: 유혹/매춘/조교)
- 이전: 해당 스킬 키 미정의
- 수정: `seduction/prostitution/training` 추가

6. 시트 가독성 미흡
- 기준: 생성 결과 점검 가능성
- 이전: 스킬 키를 내부 영문 코드로 출력
- 수정: 한국어 스킬 라벨 출력 + 시작 장비/선택 기록 섹션 추가

7. 퀘스트 본편 흐름 부재
- 기준: 3.3.3(퀘스트/이벤트 체인), 10.3(오아시스 축)
- 이전: 랜덤 이벤트만 존재, 상태 기반 퀘스트 체인 없음
- 수정: 퀘스트 로그/상태 전이 프레임워크 + 오아시스/베리타스 분기 퀘스트 추가

8. 퀘스트 전달 단계 자원 소모 버그
- 기준: 3.3.3(퀘스트 목표 충족 조건의 일관성)
- 이전: 정화수 2개 미만일 때도 일부 아이템이 먼저 소모됨
- 수정: 보유 수량 선검증 후 2개 이상일 때만 소모하도록 수정

## 현재 반영 범위

- 오프닝 선택
- 기억 편린 1~3
- 보너스 분기(밀고 시 매력/지능 선택)
- 스탯/스킬/퍽 반영
- 시작 장비 자동 할당
- 캐릭터 시트 출력 및 JSON 저장
- 생존 루프(탐험/AP/휴식/정화수 사용)
- 랜덤 인카운터(자원 발견/전투)
- 간단 전투 및 전리품 획득
- 인벤토리 열람/아이템 사용
- 기초 제작 시스템(붕대/정화수/간이 해독제)
- 세력 평판/소문 시스템 1차
- 정착지 상호작용 기반 평판 변화
- 칭호 평판 1차(황무지의 성자/약탈왕/유령)
- NPC 관계 수치 1차(신뢰/호감/공포/욕정/지배복종)
- NPC 관계 상호작용 1차(공유/위협/대화/유혹/지배)
- 퀘스트 프레임워크 1차(로그/진행도/완료 처리)
- 오아시스 수직 슬라이스 퀘스트(정화수 전달, 수호/갈취 분기, 전투 연동 완료)
- V.E.R.I.T.A.S. 수직 슬라이스 퀘스트(중계기 재료 전달, 보호/유출 분기, 전투 연동 완료)
- 난이도/밸런스 프로필 1차(스토리/스탠다드/하드코어)
- 성장 루프 1차(XP/레벨/퍼크 포인트/성장 퍼크 3종)
- UI 셸 1차(`ui/`, 저장 JSON 기반 정보 시각화 + 임시 아트 에셋 연동)
