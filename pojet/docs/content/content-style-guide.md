# Content Style Guide

## 로그 문장 규칙
- 1문장 중심, 12~28자 권장, 동사로 끝나는 리듬 유지.

## 이벤트 본문 길이
- T2: 1~2문단(짧은 맥락+갈등)
- T3: 2문단(결정의 무게 강조)

## 선택지 문장 톤
- 명령형 대신 의지형: "받는다/거절한다/보류한다"
- 결과를 노골적으로 스포일러하지 않음.

## 고유명사 작명 규칙
- 도시/세력/인물 모두 오리지널 접두/접미 규칙 사용.
- 기존 TRPG 유명명과 음운 유사도 높은 명칭 회피.

## 오리지널 세계관 유지
- 규칙 구조는 호환 감각, 사건/지명/신앙은 독자 IP.

## D&D 느낌은 살리되 복제하지 않는 작법
- 수치/판정 감각만 차용, 문장과 설정은 완전 재창작.

## 반복 이벤트 변주법
- 동일 이벤트라도 주체/장소/감정 단어를 교체하는 템플릿 사용.

## 판타지 어휘 레이어
- 현대 슬랭 대신 세계관 어휘 우선: 의식, 맹세, 제단, 장막, 낙인, 계약, 파문.
- 신체/감정 표현은 직접 명사보다 반응 중심: 떨림, 숨 고르기, 시선 회피, 침묵 후 반응.
- 같은 장면에서도 어휘 등급을 나눠 유지: 공적 톤(중립) -> 사적 톤(긴장) -> 위기 톤(대가).

## 장면 연출 우선순위
- 1순위: 상황과 이해관계(누가 무엇을 잃거나 얻는지).
- 2순위: 감정/관계 축 변화(욕망, 지배, 의존, 평판, 오염).
- 3순위: 감각 디테일(환경 소리, 조명, 거리감).
- 4순위: 후폭풍 예고(다음 틱/다음 Act에 남는 흔적).

## 금지
- 지나치게 현대적 밈, 인터넷 슬랭, 메타 농담 남용 금지.
- 코드/스키마에 없는 임의 필드 남발 금지.
- 런타임 본문에 맥락 없는 자극 문장 단독 삽입 금지.

## 이 문서가 실제 게임 기획/UX에 어떻게 쓰이는지
- 신규 콘텐츠 PR 검수 기준
- 라이팅 팀 통일 스타일 가이드
