# 00. 바로쓰기 치트시트 (바이브 코딩)

## A. 오늘 작업 한 줄
- 목표:
- 완료조건(숫자):
- 비범위:

예시:
- 목표: 선택지 다양화
- 완료조건: 30틱 기준 고유 선택지 12개 이상
- 비범위: UI 리디자인

---

## B. 바로 복붙 프롬프트 템플릿

```txt
목표: [한 줄]
범위:
- [기능 1]
- [기능 2]
비범위:
- [제외할 것]
완료조건:
- [측정 가능한 지표 1]
- [측정 가능한 지표 2]
검증:
- 문법 체크
- 시뮬 N회
결과보고:
- 변경 파일
- 전/후 지표
```

---

## C. 로그형 게임용 품질 체크
- [ ] 조사 깨짐 없음 (이/가, 을/를, 은/는, 와/과)
- [ ] 고유명사 포함 (NPC/세력/장소)
- [ ] 장면→판정→결과 구조 유지
- [ ] 반복문장 비율 낮음
- [ ] 선택 결과가 다음 로그에 반영됨

---

## D. 빠른 지표 기준 (권장)
- 평균 로그 수: 30~45
- 조기사망률: 30% 이하
- T2 빈도: 2.5~6.5 / run
- T3 빈도: 0.8 이상 / run

---

## E. 실수 방지 5개
1. 기능/밸런스/문체를 한 번에 고치지 말 것
2. 완료조건 없는 요청으로 코딩 시작하지 말 것
3. 시뮬 결과 없이 체감으로만 판단하지 말 것
4. 대형 수정 후 저장/로드 호환 체크할 것
5. 최종 반영 전 커밋 메시지에 의도 적을 것
